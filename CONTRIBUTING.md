Contribution Guidelines
=========================

Please keep each pull request focused on a single issue.

If you want to improve or fix multiple unrelated items, please open separate pull requests for each one.

## Commit Messages



## Unit Tests
Pull requests should include relevant unit tests where practical.

If your change fixes a bug, please add a test that would have failed before the fix. If your change adds or alters behaviour, please add tests that cover the expected behaviour.

If unit tests are not practical for a particular change, please explain why in the pull request description and include any manual testing steps you performed.

Untested changes may be closed or delayed until there is enough confidence that the contribution is correct and maintainable.

This policy primarily exists as one goal of this tool is that it will be used in CI pipelines, and we want a level of stability and confidence when updating.

## AI Policy
In alignment with Codebase's terms of service, we are outright banning the use of LLM generated code.
If your pr or issue has a clear indication of it being raised by an LLM, you may be blocked from future contribution.

We are not here to review the output of prompts. We are here to build and maintain quality open-source software.
